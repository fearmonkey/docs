Email templates define the content and layout used in  the authorization emails that OwnID sends automatically to users. 

You can edit the templates to customize email content so it reflects your brand and to personalize content for your customers in any language.

## About Email Templates

The OwnID platform sends emails automatically when it needs to validate users with either a <Tooltip tip="one-time passcode">OTP</Tooltip>
or a <Tooltip tip="a one-time link that grants access with a single click">magic link</Tooltip>.

For example, OwnID sends an authorization email when a user signs up or logs in for the first time but doesn't have available biometrics on their device.

### Templates for Each App
For each of your OwnID apps, you can create a unique email template to use when the system sends a _Magic Link_ email and another template to use for _Authorization(OTP)_ emails. 

## Customize your Email Templates

1. Log into the [Console](https://console.ownid.com/home) and select the App that you want to associate with your template. The My App view opens.

1. Go to _Branding > Email templates_ from the left side navigation panel. <img src="/images/oi-console-branding.png" width="200"  alt="Branding link in the Console navigation panel"></img>


1. The _Email templates_ page opens. From the _Select a template_ option, you can select the type to edit, _Magic link_ or _Verification Email (using code)_. <img src="/images/oi-et-main.png" width="400"  alt="Main email templates view showing selection of template type and language"></img>

Next, customize either or both template types for your App.

### Customize the Verification Template
1. To customize the body of your OTP verification emails, select the _Verification_ option.

1. Select the language to use in the email text from the _Language_ dropdown.

1. Enter the desired subject text. <img src="/images/oi-et-otp.png" width="500"  alt="Preview"></img>


1. Modify the HTML content directly as desired in the embedded code editor. 

All text inside the `<body>` tags can be customized, and the recipient's OTP is inserted wherever you place the `{code}` token.

```html Default OTP Email Body
<!DOCTYPE html>
<html>

  <body>
    <div>Your verification code is: {code}</div>
    <div>If you didn't request this code or you received this message by accident, please disregard this email.</div>
  </body>

</html>
```
<sup> Embedded code editor for the verification email</sup>

4. Click **Preview** to check what recipients will see in an authorization email.   <img src="/images/oi-et-preview_ml.png" width="600"  alt="Preview of rendered template for verification email"></img>
(Click **Reset** to start over with the default HTML.)
1. Click **Save**


### Customize the Magic Link Template

1. To customize the body of your magic link emails, select _Magic Link_ from the _Select a template_  dropdown.  <img src="/images/oi-et-main.png" width="400"  alt="Main email templates view"></img>
1. Select the language to use in the email text from the _Language_ dropdown.

1. Enter the desired subject text. You can embed your App name in the subject using the `{app_name}` token. <img src="/images/oi-et-ml.png" width="500"  alt="Preview"></img>

1. Modify the HTML content directly as desired in the embedded code editor. 

All text inside the `<body>` and `<style>` tags can be customized, and the recipient's magic link is inserted in the DOM wherever you place the `{link}` token.

```html Default Magic Link Email Body
<!DOCTYPE html>
<html>

  <head>
    <style>
      body {
        font-family: 72;
        font-style: normal;
        font-weight: normal;
        font-size: 14px;
        line-height: 24px;
      }

      div {
        margin-bottom: 20px;
      }

      .rounded-link {
        background: #0070F2;
        color: #FFFFFF !important;
        display: block;
        padding: 8px 26px;
        text-align: center;
        text-decoration: none;
        border-radius: 4px;
        font-weight: bold;
        width: auto;
      }
    </style>
  </head>

  <body>
    <div>Hi,<br>Click the link below to log in to your account.</div>
    <div style="display: flex"><a target="_blank" class="rounded-link" href="{link}">Your Magic Link</a></div>
    <div>For security reasons, this link will only be valid in the same browser that initiated this request. If the link above doesn't work, try to copy and paste the following URL in the browser:<br />{link}</div>
    <div>If you didnâ€™t make this magic link request, please disregard this email.</div>
  </body>

</html>
```
<sup> Embedded code editor for the magic link email</sup>


5. Click **Preview** to check what recipients will see when they get a magic link email.   <img src="/images/oi-et-preview_ml.png" width="600"  alt="Preview"></img>
1. Click **Save** to enable your new template.

### Testing
Best practice is to confirm the email functionality by signing up a new user and logging in from a device without biometric capabities.
