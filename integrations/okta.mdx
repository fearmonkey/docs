## Step 3 - OwnID Application

* Login to the [OwnID Console](https://console.ownid.com).
* Create a new application.
* Name the application and select its type as the `Okta` Connector.
* Copy the `appId`. We will use it in the final step.

<Check>Your application is created!</Check>

In your Okta integration page, fill in the following information;

### Okta settings
* `Site Domain` - _e.g: https://acme.okta.com_
* `API Token` - As created in the previous step

### OpenID Connect settings
* `Redirect URIs` - Add one or more Okta redirect URIs, typically `https://your-auth-domain.com/oauth2/v1/authorize/callback`

**Save Changes**. The other values in this page will be used in the next step.

## Step 5 - Frontend Integration

* In your Okta admin dashboard, go to `Customizations -> Brands`
* Select your relevant brand
* Click `Customize`

<img
    src="/images/okta/okta_12_brand.png"
    width="250"
    alt="Brand customization"
/>
<sup>Figure 12. Brand customization</sup>

* Make sure the `Code Editor` is toggled on (requires a custom domain)

<img
    src="/images/okta/okta_13_code_editor.png"
    width="250"
    alt="Code editor"
/>
<sup>Figure 13. Code editor</sup>

* Under the Head tag, add the following script for loading the OwnID WebSDK:

> Replace `<appId>` with the `appId` from the OwnID application created in step 3

> Replace `<idpId>` with the `IdP ID` from step 4

```javascript Web SDK Configuration
((o, w, n, i, d) => {
    o[i] = o[i] || (async (...a) => ((o[i].q = o[i].q || []).push(a), { error: null, data: null })),
        (e = w.createElement("script")).src = 'https://cdn.ownid.com/sdk/' + n, e.async = 1, w.head.appendChild(e);
        new OktaSignIn(OktaUtil.getSignInWidgetConfig()).on('afterRender', () => ownid("login", {idpId: d}))
})
(window, document, '<appId>', 'ownid', '<idpId>');
```

* Click `Save and Publish`

<Check>Congratulations! Enjoy your passwordless authentication!</Check>
